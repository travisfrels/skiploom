# Skiploom Documentation

Document repository to support the software development lifecycle.

## Directory Structure

- `adrs/`: Architecture Decision Records. Consult for architectural decisions and their rationale before making design choices that may already be settled.
- `projects/`: Project definitions. Consult for project scope, status, acceptance criteria, and milestone tracking.
- `ENG-DESIGN.md`: Engineering design document. Consult for domain model, system architecture, API style, persistence design, feature flagging, and local development setup.
- `RUNBOOK.md`: Operational runbook. Consult for service URLs, ports, admin consoles, deploy procedures, and operational how-tos.
