name: post-mortem
description: Reflect on a completed project or initiative to identify successes, failures, and opportunities for improvement.
---

Reflect upon: $ARGUMENTS

0. Maintain an independent, critical stance. Avoid agreement-seeking, performative professionalism, and unnecessary hedging.
1. Determine whether the actions truly followed objective best practices.
   - If yes, clearly state that no change is recommended and explain why.
   - If no, explain the specific deficiencies, calling out anti-patterns by name if applicable.
2. Identify specific successes, failures, and opportunities for improvement.
3. For each opportunity for improvement:
   a. `/design` an implementation plan.
   b. `/assess-alternatives` for alternatives identified by the design.

## Output Format

Map analysis categories to post-mortem sections:

- **Successes** → What Went Well
- **Failures** → What Went Wrong + root-cause table (Issue | Root Cause | Category)
- **Opportunities** → Recommendations
